2. Información que Recopilamos

2.1 Información de Notas

  • Contenido de notas: Texto, títulos y descripción de las notas que usted cree
  • Fechas: Fecha de creación y modificación de notas
  • Organización: Categorías, etiquetas o carpetas que asigne a sus notas
  • Imágenes: Fotografías o imágenes que adjunte a sus notas (almacenadas en Firebase Storage)

2.2 Información de Cuenta

  • Información de autenticación: Email y datos de autenticación proporcionados por Firebase Authentication
  • Identificador único: ID de usuario generado por Firebase
  • Preferencias: Configuraciones de la aplicación y preferencias de usuario

2.3 Información Técnica

  • Información del dispositivo: Modelo, sistema operativo, versión de la aplicación
  • Datos de uso: Funciones utilizadas, tiempo de uso, interacciones con la aplicación
  • Información de red: Dirección IP, datos de conectividad

3. Cómo Utilizamos su Información

3.1 Propósitos Principales

  • Gestión de notas: Almacenar, organizar, sincronizar y mostrar sus notas personales
  • Funcionalidad de la app: Proporcionar todas las características y servicios de la aplicación
  • Sincronización: Mantener sus datos actualizados entre dispositivos
  • Respaldo: Proteger sus datos mediante copias de seguridad automáticas

3.2 Propósitos Secundarios

  • Mejora del servicio: Analizar el uso para mejorar la funcionalidad
  • Soporte técnico: Resolver problemas y brindar asistencia
  • Seguridad: Proteger la aplicación y prevenir uso indebido

4. Almacenamiento con Firebase

4.1 Servicios de Firebase Utilizados

  • Firebase Authentication: Para el inicio de sesión seguro y gestión de usuarios
  • Firebase Database (Firestore): Para almacenamiento del contenido de las notas y metadatos
  • Firebase Storage: Para almacenamiento seguro de imágenes adjuntas a las notas
  • Firebase Analytics: Para análisis de uso básico (datos anonimizados)

4.2 Seguridad en Firebase

  • Encriptación: Todos los datos se encriptan en tránsito y en reposo
  • Reglas de seguridad: Acceso restringido solo al usuario propietario de los datos
  • Ubicación de datos: Servidores de Google Firebase
  • Cumplimiento: Firebase cumple con estándares internacionales de seguridad

4.3 Estructura de Datos

  • Database: Contenido de texto, títulos, fechas, metadatos y configuraciones
  • Storage: Archivos de imagen con nombres únicos y metadatos asociados
  • Separación por usuario: Cada usuario solo puede acceder a sus propios datos

5. Restricciones Importantes de Contenido

IMPORTANTE - Tipos de Información que NO debe ingresar:

5.1 Información Prohibida

  • Información financiera: Números de tarjetas de crédito, cuentas bancarias, PINs
  • Credenciales de acceso: Contraseñas, códigos de acceso, claves de seguridad
  • Documentos oficiales: Números de identificación, pasaportes, licencias
  • Información médica sensible: Diagnósticos, tratamientos, datos de salud críticos
  • Información laboral confidencial: Secretos comerciales, información propietaria
  • Datos personales de terceros: Información privada de otras personas sin su consentimiento

5.2 Uso Recomendado

  • Notas generales: Recordatorios, listas de tareas, ideas creativas
  • Información educativa: Apuntes de estudio, resúmenes de libros
  • Planificación personal: Planes de viaje, proyectos personales, objetivos
  • Contenido creativo: Borradores, inspiración, bocetos textuales

6. Protección de Datos

6.1 Medidas Técnicas de Seguridad

  • Encriptación:
    • HTTPS/TLS para datos en tránsito
    • Encriptación automática de Firebase para datos en reposo
  • Autenticación:
    • Sistema de autenticación robusto de Firebase
    • Tokens de sesión seguros
    • Verificación de email cuando sea requerida
  • Control de acceso:
    • Acceso exclusivo del usuario a sus propios datos
    • Reglas de seguridad de Firebase configuradas apropiadamente

6.2 Limitaciones de Seguridad

Advertencia: Aunque implementamos medidas de seguridad robustas, ningún sistema es 100% seguro. Por esta razón es fundamental que NO ingrese información sensible o confidencial en la aplicación.

7. Compartir Información

7.1 No Compartimos con Terceros

Política estricta: No vendemos, alquilamos o compartimos sus notas personales con terceros para fines comerciales.

7.2 Excepciones Limitadas

Podemos compartir información solo en estas circunstancias específicas:

  • Consentimiento explícito: Cuando usted nos autoriza expresamente
  • Proveedores de servicio: Google Firebase para el funcionamiento de la aplicación
  • Cumplimiento legal: Cuando sea requerido por ley o autoridades competentes
  • Protección de derechos: Para proteger nuestros derechos legales o los de otros usuarios

8. Sus Derechos como Usuario

8.1 Derechos de Acceso y Control

  • Acceso: Puede ver toda su información a través de la aplicación
  • Edición: Puede modificar o eliminar sus notas en cualquier momento
  • Eliminación de cuenta: Puede eliminar su cuenta y todos los datos asociados
  • Exportación: Puede exportar sus datos en formato legible
  • Restricción: Puede limitar cómo procesamos sus datos

8.2 Cómo Ejercer sus Derechos

  • Desde la aplicación: Use las opciones de configuración y gestión de datos
  • Contacto directo: Envíe una solicitud a nuestro equipo de soporte
  • Tiempo de respuesta: Responderemos dentro de 30 días

9. Retención y Eliminación de Datos

9.1 Períodos de Retención

  • Datos de notas activas: Mientras mantenga su cuenta activa
  • Cuenta inactiva: Los datos se eliminan después de 2 años de inactividad
  • Datos de soporte: Se eliminan después de 1 año
  • Logs técnicos: Se mantienen por 6 meses para seguridad

9.2 Eliminación Segura

  • Eliminación inmediata: Cuando elimina notas, se borran de inmediato
  • Eliminación de cuenta: Todos los datos se eliminan permanentemente al cerrar la cuenta
  • Copias de respaldo: También se eliminan de copias de seguridad

10. Transferencias Internacionales

10.1 Ubicación de Datos

  • Servidores principales: Google Firebase (múltiples regiones)
  • Transferencias: Sus datos pueden procesarse en diferentes países
  • Protecciones: Firebase implementa protecciones adecuadas para transferencias internacionales

11.2 Protección de Menores
